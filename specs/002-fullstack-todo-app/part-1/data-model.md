# Data Model: Full-Stack Todo App — Part 1

**Date**: 2026-03-25 | **Database**: Neon Serverless PostgreSQL

## Overview

Two systems manage tables in the same Neon database:
1. **Better Auth** (Kysely adapter) → `user`, `session`, `account`, `verification` tables
2. **SQLModel** (Alembic migrations) → `task` table

No foreign key constraint between systems — user isolation is enforced at the application layer via JWT user_id.

## Application Tables (SQLModel + Alembic)

### task

| Field | Type | Constraints | Notes |
|-------|------|-------------|-------|
| id | UUID | PK, DEFAULT uuid4 | Application-generated |
| user_id | string | NOT NULL, INDEX | Better Auth user ID (from JWT `sub` claim) |
| title | string(200) | NOT NULL | 1–200 chars (FR-005, FR-006) |
| description | string(1000) | NULLABLE | Max 1,000 chars (FR-005, FR-006) |
| is_completed | boolean | NOT NULL, DEFAULT false | Toggle via PATCH (FR-010) |
| created_at | timestamp(tz) | NOT NULL, DEFAULT now() | UTC, auto-set (FR-018) |
| updated_at | timestamp(tz) | NOT NULL, DEFAULT now() | UTC, auto-updated (FR-018) |

**Indexes**:
- `ix_task_user_id` on `user_id` — all queries filter by user_id for isolation (FR-013)
- `ix_task_created_at` on `created_at` — sort by newest first (FR-021)

### SQLModel Definition

```python
from datetime import datetime, timezone
from uuid import UUID, uuid4
from typing import Optional
from sqlmodel import SQLModel, Field

class Task(SQLModel, table=True):
    __tablename__ = "task"

    id: UUID = Field(default_factory=uuid4, primary_key=True)
    user_id: str = Field(index=True, nullable=False)
    title: str = Field(max_length=200, nullable=False)
    description: Optional[str] = Field(default=None, max_length=1000)
    is_completed: bool = Field(default=False, nullable=False)
    created_at: datetime = Field(
        default_factory=lambda: datetime.now(timezone.utc),
        nullable=False,
    )
    updated_at: datetime = Field(
        default_factory=lambda: datetime.now(timezone.utc),
        nullable=False,
    )
```

## Entity Relationships

```text
Better Auth user (id: string)
    │
    │  referenced by (no FK constraint — app-layer enforcement)
    │
    ▼
task.user_id (string)
    │
    └── task.id (UUID) ── one user has many tasks
```

**User isolation rule**: Every SQL query that reads, updates, or deletes tasks MUST include `WHERE user_id = :authenticated_user_id`. This is enforced by the FastAPI JWT middleware dependency — endpoints receive `user_id` from the verified token and pass it to all database operations.

## Validation Rules

### Authentication Inputs (FR-022)

| Field | Frontend (Zod) | Backend (Better Auth) |
|-------|----------------|----------------------|
| Display Name | `z.string().trim().min(1).max(100)` | Better Auth config: min name length |
| Email | `z.string().email()` | Better Auth built-in |
| Password | `z.string().min(8)` | Better Auth config: min password length |

### Task Inputs (FR-006 — created in Part 1, used in Part 2)

| Field | Frontend (Zod) | Backend (Pydantic) |
|-------|----------------|-------------------|
| Title | `z.string().trim().min(1).max(200)` | `Field(min_length=1, max_length=200)` + whitespace check |
| Description | `z.string().max(1000).optional()` | `Field(max_length=1000, default=None)` |

## State Transitions

### Task Status

```text
[Created] ──→ is_completed = false (default)
    │
    ▼
[Toggled] ──→ is_completed = true
    │
    ▼
[Toggled] ──→ is_completed = false (revertible)
```

No terminal state — tasks can be toggled indefinitely until deleted.
