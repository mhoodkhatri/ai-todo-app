# Implementation Plan: Task CRUD Operations (Part 2)

**Branch**: `002-fullstack-todo-app` | **Date**: 2026-03-25 | **Spec**: [spec.md](../spec.md)
**Input**: Feature specification from `specs/002-fullstack-todo-app/spec.md` — Part 2 scope per [phase-breakdown.md](../phase-breakdown.md)

**Note**: This plan covers Part 2 only — Task CRUD Operations (full-stack). Part 1 (auth, scaffold) is already implemented.

## Summary

Implement full task CRUD operations across the FastAPI backend and Next.js frontend. The backend gains 6 RESTful endpoints (create, list, get, update, toggle, delete) with user isolation and dual validation. The frontend gains a task dashboard with create form, task list view, inline editing, completion toggle, delete with confirmation, and empty state. All operations are scoped to the authenticated user via JWT. The existing Task model and database migration from Part 1 are reused as-is.

## Technical Context

**Language/Version**: TypeScript 5.x (Next.js 16+), Python 3.13+ (FastAPI)
**Primary Dependencies**:
- Backend: FastAPI, SQLModel, asyncpg, PyJWT, Alembic (all installed in Part 1)
- Frontend: Next.js 16.2.1, React 19, Tailwind CSS 4, Better Auth 1.5.6, Zod 4.3 (all installed in Part 1)
**Storage**: Neon Serverless PostgreSQL — `task` table already created (Part 1 migration `4d90e771f885`)
**Testing**: Manual acceptance testing against spec scenarios (no test framework added yet)
**Target Platform**: Web — desktop (1024px+) and mobile (320px+)
**Project Type**: Web application (monorepo: `frontend/` + `backend/`)
**Performance Goals**: Task operations complete in under 30 seconds each (SC-002); list renders within 3 seconds (SC-008)
**Constraints**: 100% user isolation (SC-003); JWT required on all API calls; last-write-wins concurrency
**Scale/Scope**: Phase II hackathon scope — no pagination, no task limits, single Neon DB
